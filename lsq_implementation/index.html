LSQ: How to implement

Consider the data and model from Simple-example:

X = collect(1:10)
Y = [1.0, 1.78, 3.64, 3.72, 5.33, 2.73, 7.52, 9.19, 6.73, 8.95]
ΔY = [0.38, 0.86, 0.29, 0.45, 0.66, 2.46, 0.39, 0.45, 1.62, 1.54]
data = FittingData(X,Y,ΔY)
model = ModelFunctions((x,λ)-> λ*x)

LSQ: objective functions

The uncertainty-weighted least squares objective function can be obtained as follows:

weighted_lsq = lsq_objective(data,model)

lsq_objective returns a function that takes the model parameters λ as argument.

To obtain the standard least squares objective, the errors must be set to 1. Recall the shortened constructor (see FittingData):

data_no_errors = FittingData(X,Y)
standard_lsq = lsq_objective(data_no_errors,model)

LSQ: partial derivatives and gradients

Examples

Redefine model to obtain analytical derivatives (see ModelFunctions):

model = ModelFunctions((x,λ)->λ*x , partials = [(x,λ)-> x])
∂_weighted_lsq = lsq_partials(data,model)
1-element Vector{Function}:
 #17 (generic function with 1 method)
∂_weighted_lsq[1](1.1)
42.62381855699296

Note that lsq_partials returns a vector of abstract functions with λ as argument (one for each partial derivative), even in the 1-dimensional case.

∇_weighted_lsq = lsq_gradient(data,model)
#23 (generic function with 1 method)
∇_weighted_lsq([0.0],1.1)
1-element Vector{Float64}:
 42.62381855699296

On the other hand lsq_gradient directly returns the gradient function, but with a different signature (grad_vector,λ). The argument grad_vector must be a vector of appropriate type and length, that can be mutated.

Why mutation for gradient function?

In some optimization algorithms, the gradient function is called multiple times during each iteration. Mutating an array allows to reduce the memory allocation overhead of creating a new gradient array every time.

Partial derivatives

lsq_partials(data::FittingData,model::ModelFunctions)

Return the partial derivatives w.r.t. the parameters [∂_1 ob(λ),…,∂_n ob(λ)] of the least squares objective function ob(λ).

  • The partial derivatives (∂_μ m)(x,λ) of the model function must be specified in the ModelFunctions object model.

Analytical expression

  • independent data points: x_i

  • dependent data points: y_i

  • errors: Δy_i (defaulting to 1)

  • model function: m

  • partial derivatives of model function in λ: ∂_μ m

$\partial_\mu \text{lsq}(\lambda) = \sum_{i=1}^N \frac{ 2 \cdot (m(x_i,\lambda) - y_i) \cdot (\partial_\mu m)((x_i,\lambda)) }{\Delta y_i^2}$

Gradient

lsq_gradient(data::FittingData,model::ModelFunctions)

Return the gradient function grad!(gradient,λ) for the least squares objective function ob(λ). The gradient function grad! mutates (for performance) and returns the gradient. The elements of gradient do not matter, but the type and length must fit.

  • The partial derivatives (∂_μ m)(x,λ) of the model function must be specified in the ModelFunctions object model.

Analytical expression

  • independent data points: x_i

  • dependent data points: y_i

  • errors: Δy_i (defaulting to 1)

  • model function: m

  • partial derivatives of model function in λ: ∂_μ m

$\nabla \text{lsq}(\lambda) = \sum_{\mu} \sum_{i=1}^N \frac{ 2 \cdot (m(x_i,\lambda) - y_i) \cdot (\partial_\mu m)((x_i,\lambda)) }{\Delta y_i^2} \ \vec{e}_\mu$
